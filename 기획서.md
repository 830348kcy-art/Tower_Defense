## 1. 프로젝트 개요

플레이어가 **40×40 타일 맵** 위에 다양한 타워를 배치하여, 웨이브 형태로 몰려오는 적을 막고 기지(본진)를 지키는 **탑뷰 타워 디펜스 게임**입니다.

킹덤러시 시리즈를 레퍼런스로 삼아 **30개 스테이지**, **중간 보스·최종 보스**, **테크트리 영구 강화 시스템**, **특수 스킬** 을 구현하였습니다.

> **AI 활용**: 킹덤러시 위키·Steam 커뮤니티·공략 사이트 자료를 Claude AI로 사전 조사하여 타워 밸런스·적 상성·웨이브 설계에 반영하였습니다.

## 2. 핵심 게임플레이 구조

| 요소 | 킹덤러시 방식 | 본 프로젝트 적용 |
|---|---|---|
| 시점 | 탑뷰 2D | 탑뷰 2.5D (레이어 원근감) |
| 맵 구조 | 선형 경로(Path) | 40×40 타일 그리드 + Waypoint 경로 |
| 타워 설치 | 지정 슬롯 클릭 | **건설 가능 타일 클릭** (경로 외 모든 타일) |
| 타워 종류 | 4종 (아처/마법/폭격/병영) | 동일 4종 + Lv4 분기 2가지 |
| 적 웨이브 | 라운드 단위 스폰 | WaveManager 라운드 관리 |
| 조기 웨이브 호출 | 잔여시간 비례 골드 보너스 | **동일 구현** (핵심 텐션 요소) |

| 강화 시스템 | 별 소모 영구 강화 | 테크트리 10종 노드 |

## 4. 타워 시스템

### 4-1. 기본 4종

| 타워 | 공격 방식 | 특성 | 건설 비용 | 강점 | 약점 |
|---|---|---|---|---|---|
| **아처 타워** | 단일 원거리 | 빠른 공격속도 | 70G | 비행 적, 저HP 다수 | 중장갑 |
| **마법사 타워** | 단일 + 슬로우 | 마법 피해(장갑 무시) | 100G | 중장갑, 슬로우 | 마법 저항 적 |
| **폭격 타워** | 광역 스플래시 | 느린 속도, 범위 피해 | 150G | 군중, 중장갑 | 비행 불가 |
| **병영** | 근접 유닛 소환 | 적 이동 차단 | 80G | 길목 막기, 시너지 | 단독 DPS 낮음 |

### 4-2. 업그레이드 구조

```
Lv1 → (비용) → Lv2 → (비용) → Lv3 → [분기 선택]
                                         ├─ 분기 A
                                         └─ 분기 B
```

### 4-3. Lv4 분기 상세

| 타워 | 분기 A | 분기 B |
|---|---|---|
| 아처 | **사격수** — 단일 초고데미지, 긴 사거리 (250G) | **속사** — 다중 연사, 광역 견제 (250G) |
| 마법사 | **서리 마법사** — 광역 빙결 슬로우 강화 (300G) | **화염 마법사** — 도트 화염 피해 + 광역 (300G) |
| 폭격 | **박격포** — 초장거리 초광역 스플래시 (400G) | **지뢰 설치** — 즉발 광역 피해 (400G) |
| 병영 | **성기사 부대** — 고체력 탱커 3체 (280G) | **도적 부대** — 고DPS 어쌔신 3체 (280G) |

### 4-4. 타워 수치 (Lv1 기준)

| 타워 | 강함 | 약함 |
|---|---|---|

### 5-3. HP 스케일링 (스테이지별)

```
실제 HP = 기본 HP × (1 + (스테이지 번호 - 1) × 0.10) × (1 - 테크트리 적HP감소 보너스)

예) 30스테이지 보스 (테크 미적용): 4,000 × (1 + 29 × 0.10) = 4,000 × 3.9 = 15,600 HP
```

## 6. 스테이지 설계 (30스테이지)

### 6-1. 전체 구성표

| 구간 | 스테이지 | 테마 | 경로 형태 | 특징 |
|---|---|---|---|---|
| **입문** | 1 ~ 5 | 초원 평지 | 직선형 | 적 2종, 중간보스(5), 아처 시작 |
| **성장** | 6 ~ 10 | 숲 지대 | U자형 | 병영·마법 해금, 보스(10) |
| **중반** | 11 ~ 15 | 사막 협곡 | 분기 합류형 | 좁은 통로 효과, 중간보스(15) |
| **심화** | 16 ~ 20 | 화산 지형 | 복잡 굴곡형 | **야간 효과**(시야↓), 보스(20) |
| **고급** | 21 ~ 25 | 눈밭·빙하 | 3경로 동시 | **빙판 효과**(이속+15%), 중간보스(25) |
| **최종** | 26 ~ 30 | 마왕 성채 | 4중 교차형 | 최종 보스(30), 전 종류 혼합 |

### 6-2. 경로 형태 변화

```
[Stage 1~5: 직선형]
S ──────────────────── B

[Stage 6~10: U자형]
S ──↓
    ↓────────↑
             ↑──── B

[Stage 11~15: 분기 합류형]
S1 ──↓
      ↘──────── B
S2 ──↗

[Stage 16~20: 굴곡형]
S ──↓  ↑──↓
    ↓──↑  ↓── B

[Stage 21~25: 3경로 동시]
S1 ──────────────→ B
S2 ──────────────→ B
S3 ──────────────→ B

[Stage 26~30: 4중 교차형]
S1 ──→↓ ↑──→ B
S2 ──→↑ ↓──→ B
S3 ──────────→ B
S4 ──────────→ B
```

### 6-3. 환경 효과

| 효과 | 적용 구간 | 내용 |
|---|---|---|
| 좁은 통로 | 11~15 스테이지 | 건설 가능 타일 수 제한 |
| 야간 스테이지 | 16~20 스테이지 | 화면 어두워짐 (Opacity 0.30 오버레이) |
| 빙판 타일 | 21~25 스테이지 | 경로 색상 변경 + 스피드 스케일 +15% |
| 용암 타일 | 16~20 스테이지 | 일부 건설 슬롯 제한 |

### 6-4. 보스 스케일링

```
보스 HP = 기준 HP × (1 + (스테이지 - 1) × 0.10)

10스테이지 보스 HP: 4,000 × 1.9 = 7,600
20스테이지 보스 HP: 4,000 × 2.9 = 11,600
30스테이지 보스 HP: 4,000 × 3.9 = 15,600
```

## 7. 웨이브 시스템

### 7-1. WaveSpawner 구조

- 스테이지당 웨이브 수: 4~10개 (스테이지 번호에 따라 증가)
- 웨이브 자동 시작: 이전 웨이브 종료 후 `TimeUntilNext`초 후 자동 진행
- **조기 호출(Early Call)**: `[N]` 키 또는 버튼으로 다음 웨이브 즉시 호출
  - 잔여 시간 × 2 = **보너스 골드** 즉시 지급
  - 킹덤러시 핵심 텐션 메커니즘 동일 구현

### 7-2. 웨이브 자동 구성 로직

```
wavePower = 8 + 스테이지 × 1.5 + 웨이브 번호 × 2.0
SpawnInterval = max(0.35, 0.8 - 스테이지 × 0.01)

스테이지 별 등장 적:
  - 1+:  고블린 병사
  - 2+:  고블린 스카우트 추가 (60% 확률)
  - 4+:  오크 전사 추가 (40%)
  - 7+:  와이번 추가 (35%)
  - 9+:  트롤 주술사 추가 (30%)
  - 16+: 암흑 기사 추가 (40%)
  - 5 / 15 / 25: 중간 보스 웨이브 추가
  - 10 / 20 / 30: 보스 웨이브 추가
```

```
│   ■ 밝은 타일 = 마우스 호버 하이라이트                     │
│   🔺 빨간 삼각형 = 적 스폰 지점                           │
│   🏰 파란 성 = 기지                                       │
│                                                          │
├──────────────────────────────────────────────────────────┤
│ [다음 웨이브 (N)]  남은시간: 12초  │ 🔥화포[1]  🛡지원[2] │  ← 하단 스킬바
└──────────────────────────────────────────────────────────┘
```

```

┌───────────────────┐         ┌──────────────────────┐
│   타워 건설        │         │  아처 타워 Lv2        │

│ [병영  80G]       │         │  [분기A 사격수 250G]  │
│ [마법 100G]       │         │  [분기B 속사    250G] │
│ [폭격 150G]       │         │  [판매 (105G)]        │
│ [닫기]            │         │  [닫기]               │
└───────────────────┘         └──────────────────────┘
```

### 조작 방법

| 키/마우스 | 동작 |
|---|---|
| 좌클릭 (건설 타일) | 타워 선택 팝업 |

## 12. 기술 구현

### 12-1. 핵심 시스템

| 시스템 | 구현 방식 |
|---|---|
| 게임 루프 | `DispatcherTimer` 16ms (60fps) |
| 렌더링 | WPF `Canvas` + 런타임 `Shape` 객체 직접 조작 |
| 맵 타일 | 40×40 그리드, 경로 판별은 `DistPointSeg()` 함수 |
| 적 이동 | Waypoint 기반 선형 보간 |
| 투사체 | 호밍(단일 타겟) / 탄도(Ballistic, 광역) 분리 |
| 병영 유닛 | 독립 AI (타겟 탐색 → 교전 → 리스폰) |
| 세이브 | `System.Text.Json` → `%LOCALAPPDATA%\KingdomRushClone\save.json` |

### 12-2. 디자인 패턴

| 패턴 | 적용 위치 |
|---|---|
| **Singleton** | `EconomyManager` (Gold 전역 관리) |
| **Observer** | `GameEngine.StateChanged` 이벤트 → HUD 자동 갱신 |
| **State** | 보스 페이즈 전환 상태 머신 |
| **Command** | 타워 건설/업그레이드/판매 (Undo 확장 가능) |
| **Strategy** | 타워 타겟팅 전략 (현재: 경로 진행도 가장 앞선 적) |

### 12-3. MVVM 구조

```
View     : XAML (GamePage, MainMenuPage, StageSelectPage, TechTreePage)
ViewModel: GameEngine.StateChanged 이벤트로 HUD 바인딩
Model    : Models/ + Data/ + Game/ 디렉터리
```

### 12-4. 기술 스택
